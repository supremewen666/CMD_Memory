# 2604.17658 - ErrorProbe: Self-Improving Error Diagnosis in Multi-Agent Systems

- **arXiv**: 2604.17658
- **Core Contribution**: ErrorProbe framework for step-level failure attribution in multi-agent systems. Uses three-stage pipeline: taxonomy-driven anomaly detection → symptom-driven backward tracing → multi-agent validation team (Strategist, Investigator, Arbiter). Maintains "verified episodic memory" updated only when error patterns are confirmed by executable evidence.
- **Key Concepts**: Verified episodic memory, step-level error localization, symptom-driven backward tracing, cross-domain transfer without retraining.
- **CMD Relevance**: Independently validates CMD's Failure Memory pattern: only store repair records when evidence confirms the diagnosis. ErrorProbe's "verified episodic memory" is structurally analogous to CMD's ECS Failure Memory (store only after Post-Repair Context Replay validates the repair). Multi-agent validation team validates CMD's subagent judge as comparator, not final arbiter.
- **Open Gap**: ErrorProbe uses observational backward tracing, not counterfactual intervention. Step-level localization is finer than capability-level but still coarser than CMD's operation-level attribution.
