# 2605.17467 — VerifyMAS: Hypothesis Verification for Failure Attribution in LLM Multi-Agent Systems

- **Core contribution**: Error-first attribution via hypothesis verification against full trajectories. Instead of predicting faulty agents directly, VerifyMAS formulates and verifies failure hypotheses, decomposing attribution into trajectory-level error validation + fine-grained agent localization.
- **Key concepts**: hypothesis verification framework, error-first attribution (not agent-first), structured error taxonomy, fine-tuned LLM verifier, trajectory-level failure verification.
- **CMD relevance**: Directly competitive attribution method. VerifyMAS = agent-level attribution in multi-agent systems via hypothesis verification; CMD = operation-level attribution in memory pipeline via counterfactual replay. Different granularity, similar goal. VerifyMAS's "error taxonomy" parallels CMD's pipeline labels. Both target automated attribution but through different mechanisms (verification vs replay). Key differentiator: CMD's counterfactual replay provides causal evidence (Recovery Gain), VerifyMAS's verification is observational.
- **Open gap**: VerifyMAS targets agent-level (which agent failed), not sub-agent operation-level (which memory operation failed). CMD's operation-level granularity remains underexplored.
- **Evaluation**: Aegis-Bench, Who&When; Qwen and GPT models.
