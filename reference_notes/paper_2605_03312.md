# 2605.03312 - MemFlow: Intent-Driven Memory Orchestration for Small Language Model Agents

- **arXiv**: 2605.03312
- **Core Contribution**: Training-free memory orchestration framework that externalizes memory planning from the SLM. Uses Router Agent (intent classification) → Memory Agent (three-tier: Profile Lookup, Targeted Retrieval, Deep Reasoning) → Answer Agent → Validator Agent (evidence check, optional retry). Achieves ~2× accuracy gain on frozen Qwen3-1.7B.
- **Key Concepts**: Route-then-compile, intent-driven memory dispatch, tier-aware token budgeting, externalized memory planning.
- **CMD Relevance**: Validates `route_error` as a V1 label — MemFlow's Router is the explicit target for Oracle Route replay. Mismatched memory operations (wrong tier for query intent) are confirmed as a real failure mode. Validator Agent's evidence-check → retry loop is a lightweight analogue of CMD's Post-Repair Context Replay.
- **Open Gap**: MemFlow's Router is rule/classification-based, not counterfactually validated. CMD's Oracle Route replay could independently verify whether a routing error was the true failure cause.
