# 2605.14421 MemLineage: Lineage-Guided Enforcement

- Core: Cryptographic provenance (RFC-6962 Merkle log, Ed25519-signed entries) + LLM-mediated derivation lineage (weighted DAG); max-of-strong-edges propagation; Untrusted-Path Persistence guarantee.
- Key: Treats memory trust as chain-of-custody problem (not filtering). Six-module design. Weighted derivation DAG records which retrieved entries influenced each new memory.
- CMD relevance: Provenance infrastructure CMD currently lacks. Lineage DAG could inform CMD's influence tracking (MEMOREPAIR gap). Chain-of-custody model complements CMD's attribution: provenance tells you where evidence came from; CMD tells you which operation lost it.
- Gap: MemLineage prevents untrusted influence; CMD diagnoses failures after they occur. Complementary: MemLineage for prevention, CMD for diagnosis.
