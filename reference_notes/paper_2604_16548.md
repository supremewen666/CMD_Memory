# 2604.16548 - Survey: Security of Long-Term Memory in LLM Agents (Mnemonic Sovereignty)

- **arXiv**: 2604.16548
- **Core Contribution**: First systematic survey treating persistent agent memory as an independent security problem. Proposes 6-phase memory-lifecycle framework (Write, Store, Retrieve, Execute, Share, Forget/Rollback) × 4 security objectives (integrity, confidentiality, availability, governance). Introduces "mnemonic sovereignty" concept.
- **Key Concepts**: Memory poisoning, extraction attacks, retrieval corruption, cross-agent propagation, governance primitives, mnemonic sovereignty.
- **CMD Relevance**: Strongly validates CMD's Subagent Judge Monitor leak-safety boundary (monitor must not leak memory content through evidence pointers or anomaly_reason). Shows concentration of attacks on write/retrieve phases — exactly CMD's pipeline labels. Governance gap (no architecture covers all 9 primitives) maps to CMD's ECS repair governance.
- **Open Gap**: Survey identifies need for "verifiable, recoverable governance" — CMD-Audit's sandboxed, evidence-driven repair records are a partial answer. No existing system provides operation-level audit trails for memory integrity.
