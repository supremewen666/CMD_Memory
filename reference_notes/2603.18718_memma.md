# MemMA: Coordinating the Memory Cycle through Multi-Agent Reasoning and In-Situ Self-Evolution

arXiv: 2603.18718 (2026-03)

## Core Contribution

Plug-and-play multi-agent framework coordinating memory cycle along forward and backward paths. Forward: Meta-Thinker steers Memory Manager (construction) and Query Reasoner (retrieval with diagnosis-guided iterative refinement). Backward: in-situ self-evolving memory — synthesizes probe QA pairs after each session, verifies memory, converts failures into repair actions before memory commit.

## Key Mechanisms
- **Planner–worker architecture**: Meta-Thinker → Memory Manager + Query Reasoner
- **Diagnosis-guided iterative retrieval**: identifies missing evidence and how to retrieve it
- **In-situ self-evolution**: synthesize probes → verify → repair → commit. "Turns downstream failures into immediate, localized repair signals"
- **Evidence-grounded repair**: diagnoses whether failure reflects missing info or retrieval difficulty

## CMD Relevance

**Structurally closest to CMD's full loop among multi-agent approaches.** In-situ probe→verify→repair mirrors CMD's counterfactual replay→attribution→ECS. Key differences:
- MemMA: multi-agent, LLM-based diagnosis (observational)
- CMD: single-agent harness, counterfactual replay (causal)
- MemMA: repair before commit (preventive), CMD: repair after failure (reactive)

MemMA's in-situ self-evolution validates the "repair before propagation" pattern. Could inform CMD V2 runtime repair loop.
