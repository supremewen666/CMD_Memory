# STAR: Stage-Attributed Triage and Repair for RCA Agents in Microservices

arXiv: 2605.15581 (2026-05)

## Core Contribution

STAR frames agent failure as a **stage-localizable reasoning bug** rather than monolithic end-to-end error. Four structured stages (Evidence Package → Hypothesis Set → Analysis Structure → Decision Report), with stage-wise auditing, budget-aware Fast/Slow Routing, decisive stage localization via counterfactual candidate evaluation, and stage-specific patch-and-replay repair.

## Key Mechanisms

1. **Stage-wise audit and diagnosis**: transforms vague failure signals into explicit stage-level inconsistency evidence
2. **Fast/Slow Routing**: lightweight local repair for near-miss traces; replay-based localization for strongly contaminated cases
3. **Decisive stage localization via counterfactual candidate evaluation**: candidate stage patches assessed by replaying downstream stages — identifies the earliest stage whose correction restores RCA consistency
4. **Patch-and-replay repair**: after blaming the stage, repairs the artifact and replays only downstream stages
5. **Self-Evolving Repair Memory**: stores successful repairs for reuse

## Key Results
- Consistently improves root cause localization and fault type classification over baselines
- Identifies decisive faulty stage with high precision
- Repairs most initially incorrect traces within 1–2 replay rounds
- Built on LangGraph for node-level replay

## CMD Relevance

**Directly competitive architecture.** STAR's stage-attribution + counterfactual candidate evaluation + patch-and-replay mirrors CMD's operation-level attribution + Recovery Gain + ECS repair. The architectural convergence (decompose→attribute→patch→replay→store) suggests this pattern is generalizing across domains.

**Key differences:**
- Domain: RCA for microservices vs agent memory pipeline
- Granularity: stage-level (4 coarse stages) vs operation-level (11 fine-grained ops)
- Evidence: stage inconsistency evidence vs recovery gain on answer correctness
- STAR has explicit Fast/Slow budget routing; CMD has RPE prefilter (planned)
