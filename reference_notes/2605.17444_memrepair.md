# 2605.17444 — MemRepair: Hierarchical Memory for Agentic Repository-Level Vulnerability Repair

- **Core contribution**: Three-layer memory architecture (History-Fix, Security-Pattern, Refinement-Trajectory) with dynamic feedback-driven refinement loop for code vulnerability repair. Reuses prior fixes and learns from failed validation attempts.
- **Key concepts**: hierarchical repair memory, failure-to-success trajectory reuse, persistent repair conventions, iterative refinement loop, experience-driven repair.
- **CMD relevance**: Architectural reference for multi-layer Failure Memory. MemRepair's three layers (History-Fix / Security-Pattern / Refinement-Trajectory) parallel potential CMD Failure Memory layers (corrected_memory / repair_guidance / contrastive_context). "Failure-to-success" trajectory reuse validates CMD's ECS store-and-retrieve paradigm. Different domain (code vulnerability repair vs memory operation repair).
- **Open gap**: MemRepair targets code repair; the concept of hierarchical repair memory for agent memory operations is unexplored. CMD's Failure Memory could adopt similar hierarchical structure.
- **Evaluation**: SEC-Bench (58.0%), PatchEval (58.2%), Multi-SWE-bench C++ (30.58%), outperforms OpenHands and SWE-agent.
