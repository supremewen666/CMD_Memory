# Trajectory-Informed Memory Generation Note

- arXiv: 2603.10600
- Core contribution: automatic extraction of actionable learnings from agent execution trajectories via four components including Decision Attribution Analyzer.
- Key concepts: Trajectory Intelligence Extractor, Decision Attribution Analyzer (pinpoints which decisions caused failures/recoveries/inefficiencies), Contextual Learning Generator (strategy/recovery/optimization tips), Adaptive Memory Retrieval, 149% relative gain on complex AppWorld tasks.
- CMD relevance: the Decision Attribution Analyzer is the closest existing work to CMD's operation-level attribution. However it attributes to decisions, not to memory pipeline stages. CMD's counterfactual replay provides stronger causal evidence.
- Open gap: trajectory attribution uses observational analysis, not counterfactual intervention; CMD's replay-based Recovery Gain metric is a stricter attribution standard.
