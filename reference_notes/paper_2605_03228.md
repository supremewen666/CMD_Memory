# MAGE: Safeguarding LLM Agents against Long-Horizon Threats via Shadow Memory

arXiv: 2605.03228, 2026-05-04

Authors: Yuhui Wang, Tanqiu Jiang, Jiacheng Liang, Charles Fleming, Ting Wang

## Core Contribution

MAGE (Memory As Guardrail Enforcement) is a defensive framework against long-horizon threats to LLM agents. Inspired by the "shadow stack" abstraction in systems security, MAGE maintains a dedicated safety-focused agentic memory (shadow memory) that distills and retains safety-critical context across the agent's full execution trajectory, proactively assessing risk of pending actions before execution.

## Key Concepts

- **Shadow Memory**: A separate, safety-dedicated memory that runs parallel to the agent's normal memory, distilling safety-critical context
- **Long-horizon threats**: Attacks that exploit extended user-agent-environment interactions, improbable in single-turn settings
- **Proactive risk assessment**: Evaluates pending actions against shadow memory before execution, enabling early-stage detection
- **Negligible utility overhead**: Defense does not degrade normal agent performance

## Key Results

- Substantially outperforms existing defenses across diverse long-horizon threats in detection accuracy
- Achieves early-stage detection for the majority of attacks
- First framework to systematically detect and mitigate long-horizon agent threats via memory architecture

## CMD Relevance

1. **Shadow memory ~ Failure Memory**: MAGE's shadow memory concept is structurally analogous to CMD's Failure Memory — both maintain a purpose-built, safety/correctness-focused memory store that intervenes before errors cause harm. Difference: MAGE targets security threats, CMD targets correctness failures.

2. **Validates `safety_error` as first-class V1 label**: MAGE demonstrates that safety filtering failures (false-positive blocking of useful evidence, false-negative admission of unsafe memory) are real and exploitable. This is direct evidence supporting CMD's V1 plan to add `safety_error` with Safety-Off/Safety-Oracle replays.

3. **Proactive assessment ~ Monitor gating**: MAGE's "assess before execute" pattern mirrors CMD's Subagent Judge Monitor → replay gating: detect risk cheaply, then invoke deeper inspection.

4. **Trojan Hippo connection**: MAGE and Trojan Hippo (2605.01970) were published 1 day apart, both addressing the agent memory attack surface from opposite sides (attack vs. defense).

## Open Gap

MAGE detects safety threats but does not do operation-level attribution. A MAGE-like shadow memory could serve as CMD's `safety_error` detection layer, with counterfactual replay diagnosing whether the failure was false-positive blocking (Safety-Off fixes it) or false-negative admission (Safety-Oracle fixes it).
