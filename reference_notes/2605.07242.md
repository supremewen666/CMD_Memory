# 2605.07242 - MEMOREPAIR: Barrier-First Cascade Repair in Agentic Memory

- **Core contribution**: Formalizes cascade update problem — when a source artifact is deleted/corrected/invalidated, descendants derived from it remain visible and steer actions with stale support. MEMOREPAIR reduces invalidated-memory exposure from 69.8-94.3% to 0% with complete influence provenance.
- **Key concepts**: Barrier-first cascade repair; influence edges (provenance) vs semantic edges (retrieval); predecessor-closed successors; s-t min-cut for repair-cost optimization.
- **Artifact types**: summaries, cached outputs, embeddings, learned skills, executable tool procedures, chain procedures, neural skills.
- **Four-phase method**:
  1. **Cascade Localization & Withdrawal Barrier**: Compute affected cascade C(F) via influence-edge reachability from invalidated root set. Withdraw all C(F) from service — no stale descendant remains visible during repair.
  2. **Artifact-Aware Candidate Generation**: Mode map assigns repair operator per descendant: remove, recompute (replay from retained support), regen (rebuild summaries/skills), param (repair neural parameters). Root replacements and interface updates supplied via Ret(e) and κe.
  3. **Cost-Aware Publication Selection**: Solve max-weight predecessor closure → s-t min-cut. For repair-cost tradeoff λ, select validated successors maximizing (weight - λ·cost) subject to predecessor-closure constraint.
  4. **Execute, Validate & Republish**: Validation checks schema validity, execution compatibility, task/safety behavior, forget/reference behavior for neural skills. Only validated predecessor-closed successors re-enter service.
- **Key equation**: max Σ w_i·x_i - λ·Σ c_i·x_i subject to executability and predecessor-closure.
- **Results**: 0% invalidated-memory exposure on ToolBench and MemoryArena (vs 92.4-99.7% post-event action dependency on invalidated info without cascade repair). Recovers 91.1-94.3% validated successors at 0.57-0.76 normalized repair cost vs exhaustive Repair all.
- **CMD relevance**: CMD attributes failure to specific memory operations → MEMOREPAIR handles downstream cascade effects when repairing those operations. CMD's Post-Repair Context Replay is a natural validation gate for MEMOREPAIR's successor republication (checks task/safety behavior, semantic quality). The influence-edge provenance tracking is infrastructure CMD currently lacks — critical for V1/V2 cascade-aware repair.
- **Open gap**: Requires complete influence provenance. No integration with automated failure attribution (CMD's role). Validation checks are schema/task/safety — no semantic quality gate equivalent to Post-Repair Context Replay.
