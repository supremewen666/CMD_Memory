# 2605.15000 Premature Closure in Frontier LLMs

- Core: Defines LLM premature closure as inappropriate commitment under uncertainty — answering when the safer response is clarification, abstention, escalation, or refusal. Evaluated 5 frontier LLMs on medical tasks.
- Key: 55-81% false-action rate on MedQA with correct answer removed. Safety prompting reduces but doesn't eliminate. "Residual failure highlights need to evaluate whether LLMs know when not to answer."
- CMD relevance: Maps to `reasoning_error` — agent reasons over incomplete/missing evidence. CMD's Evidence-Given Reasoning replay detects premature closure by giving agent the evidence and measuring recovery.
- Gap: Premature closure focuses on when models should abstain; CMD focuses on why a wrong answer was given. Complementary: premature closure = don't answer; CMD = fix the answer.
