# RSCB-MC: Risk-Sensitive Memory Retrieval Note

- arXiv: 2604.27283
- Core contribution: reframes memory use in coding agents as risk-sensitive control (contextual bandit), where the key question is not similarity but safety; abstention is a first-class action.
- Key concepts: 16-feature contextual state (relevance, uncertainty, risk, false-positive cost), asymmetric reward penalizing unsafe injection more than missed reuse, pattern-variant-episode schema, 0.0% false-positive rate, 331μs p95 latency.
- CMD relevance: directly informs retrieval_error attribution boundary — CMD should distinguish "retrieved wrong memory" from "retrieved right memory but injected unsafely." RSCB-MC's abstention action is the production analogue of CMD's "retrieval did not cause failure" verdict.
- Open gap: RSCB-MC is retrieval-only; does not diagnose upstream write/compression/extraction errors that degrade retrieval candidate quality.
