# 2604.15774 - MemEvoBench: Benchmarking Memory MisEvolution in LLM Agents

- **arXiv**: 2604.15774
- **Core Contribution**: First benchmark for "memory misevolution" — gradual behavioral drift from repeated exposure to misleading memory. Covers 7 domains, 36 risk types, plus workflow tasks from 20 Agent-SafetyBench environments.
- **Key Concepts**: Memory misevolution, adversarial memory injection, noisy tool outputs, biased feedback, mixed benign/malicious memory pools.
- **CMD Relevance**: Directly validates V1 `item_poisoned` and `item_stale` labels. Shows static prompt-based defenses are insufficient against evolving memory contamination. CMD's ECS repair loop addresses the exact gap identified: ongoing memory correction over multi-turn interactions.
- **Open Gap**: Benchmark detects mis-evolution but does not attribute which memory operation or item caused it. CMD's counterfactual replay could complement MemEvoBench by providing operation-level root cause.
