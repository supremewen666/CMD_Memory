# A-MemGuard: Proactive Memory Defense Note

- arXiv: 2510.02373
- Core contribution: first proactive defense framework for LLM agent memory; consensus-based validation across reasoning paths + dual-memory structure storing "lessons" from detected failures.
- Key concepts: self-checking memory, self-correcting memory, cross-referencing reasoning paths, separate lesson memory, breaks self-reinforcing error cycles, 95% attack success reduction.
- CMD relevance: A-MemGuard's dual-memory (active memory + lesson memory) is a production analogue to CMD's Failure Memory. The consensus-based anomaly detection parallels Subagent Judge Monitor but focuses on security rather than task failure.
- Open gap: A-MemGuard targets adversarial injection only; does not cover write/compression/extraction/retrieval/reasoning errors that CMD addresses.
