# 2605.17439 — DiagEval: Trajectory-Conditioned Diagnosis for Reliable Software Evaluation with GUI Agents

- **Core contribution**: Trajectory-conditioned diagnostic evaluation protocol that reuses failed trajectories to choose targeted diagnostic probes, aggregating outcomes into internal attribution signals. Recovers 45.6-62.1% of initially misattributed failures.
- **Key concepts**: trajectory-conditioned diagnosis, diagnostic probes, failure attribution disambiguation, latent UI state-transition graph.
- **CMD relevance**: Methodological parallel: both reuse failure trajectories for diagnosis. DiagEval reuses the failed GUI trajectory to disambiguate evaluator error vs software defect; CMD replays the failed task with counterfactual memory interventions to attribute which operation failed. Both are "failure trajectory → targeted probes → attribution" pipelines.
- **Open gap**: DiagEval targets GUI software evaluation; the trajectory-conditioned probe concept for memory operation diagnosis is novel to CMD.
- **Evaluation**: WebDevJudge-Unit (69.9→78.3%), RealDevBench (65.0→81.6%), 34.4-160.6% relative gains over retry baselines.
