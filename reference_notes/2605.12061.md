# 2605.12061 - SAGE: Self-Evolving Agentic Graph-Memory Engine

- **Core contribution**: Dynamic graph memory with two coupled roles — memory writer incrementally constructs structured graph from interaction histories; Graph Foundation Model-based memory reader performs retrieval and provides feedback to writer. Self-evolution through reader→writer feedback loop.
- **Key concepts**: Self-evolving graph memory; reader-writer coupling; graph foundation model; evidence recovery; multi-hop QA optimization.
- **Results**: After 2 self-evolution rounds, best average rank on multi-hop QA. Zero-shot open-domain: 82.5/91.6 Recall@2/5 on NQ. Improves multiple long-term memory and hallucination-diagnostic metrics on LongMemEval and HaluMem.
- **CMD relevance**: SAGE's reader→writer feedback loop is structurally similar to CMD's detect→diagnose→store→reuse loop but at different abstraction level — SAGE optimizes graph structure, CMD attributes operation-level failures. SAGE's self-evolution could generate probe cases for CMD: memory operations that were once good but degraded.
- **Open gap**: No failure diagnosis when self-evolution itself introduces errors. Feedback-driven graph mutation without validation gate — similar to consolidation risk in 2605.12978.
