# Thought-Retriever: Self-Evolving Memory Note

- arXiv: 2604.12231
- Core contribution: model-agnostic algorithm using LLM's intermediate responses (thoughts) as long-term memory; self-evolves through continuous interaction without fine-tuning.
- Key concepts: thought memory (filtered intermediate responses), self-evolving long-term memory, deeper thoughts for abstract queries, AcademicEval benchmark, 7.6% F1 gain, 16% win rate gain.
- CMD relevance: validates that intermediate reasoning artifacts carry recoverable evidence — supports CMD's Verbatim Event Oracle premise that raw events contain evidence lost during extraction. Self-evolution pattern parallels CMD's ECS-driven memory improvement loop.
- Open gap: Thought-Retriever's "thoughts" are one type of intermediate artifact; CMD generalizes to any memory operation's input/output trace for attribution.
