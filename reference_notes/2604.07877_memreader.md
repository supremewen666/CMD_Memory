# MemReader: From Passive to Active Extraction for Long-Term Agent Memory

arXiv: 2604.07877 (2026-04)

## Core Contribution

Reframes memory extraction from passive transcription to active memory management. MemReader-4B uses ReAct-style paradigm: evaluates information value, reference ambiguity, and completeness before acting — can write, defer, retrieve history, or discard.

## Key Findings
- Passive extraction = root cause of memory pollution, low-value writes, inconsistency
- Active decision-making (judge value → check completeness → retrieve history → decide) outperforms passive
- SOTA on LOCOMO, LongMemEval, HaluMem for knowledge updating, temporal reasoning, hallucination reduction
- GRPO with multi-dimensional reward constrains reasoning length to prevent hallucination

## CMD Relevance

Validates that memory extraction quality is a root cause of downstream failures — supports CMD's `write_error` and `compression_error` labels. MemReader's active extraction is the preventive complement to CMD's retrospective diagnosis.
