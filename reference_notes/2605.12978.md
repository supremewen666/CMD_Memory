# 2605.12978 - Useful Memories Become Faulty When Continuously Updated by LLMs

- **Core contribution**: Consolidated memories produced by LLMs degrade under continuous updates — memory utility first rises then falls below no-memory baseline. Even consolidating from ground-truth solutions, GPT-5.4 fails on 54% of ARC-AGI problems it previously solved without memory.
- **Key concepts**: Consolidation-induced regression; episodic-only control competitive with consolidators; consolidation gating needed; raw episodes as first-class evidence; Complementary Learning Systems (CLS) dual-store architecture.
- **Three failure modes identified**:
  1. **Misclassification/Mis-segmentation**: LLM merges episodes across problem-class boundaries during abstraction, creating over-generalized schemas. Under forced consolidation, the model routinely merges across class boundaries it otherwise recognizes.
  2. **Interference**: Each abstraction pass smooths and abstracts existing entries; early abstractions anchor later rewrites. Cumulative consolidation accumulates over-generalized memories at ~5× Fresh's rate and garbage at ~20×.
  3. **Overfit**: Repeated consolidation from narrow input distribution causes memory to overfit to specific instances, collapsing on variants.
- **Experimental design**: ARC-AGI Stream testbed with three control loops — Force (must consolidate each round), Auto (agent chooses Retain/Delete/Consolidate), Episodic Management Only (no abstraction allowed). Two stores: Episodic buffer + Abstract store.
- **Key findings**: (1) Consolidation fragility: same trajectories yield qualitatively different memories under different schedules. (2) Grouping by task family helps; one-shot beats streaming; heterogeneous batches accelerate decline. (3) Agents choose episodic-first policy when given autonomy: saturate episodic buffer quickly, keep abstract store sparse. (4) Episodic Management Only matches or exceeds forced consolidation. (5) Forced abstraction underperforms retained-episode policies across 400 training steps.
- **Design principle**: Episodic and schema-forming roles should not be collapsed into a single rewrite loop. Consolidation should be gated explicitly rather than fired after every interaction.
- **CMD relevance**: Directly validates `compression_error` and `premature_extraction_error` — consolidation degrades evidence CMD's counterfactual replay depends on. The three failure modes map to CMD labels (misclassification→premature_extraction, interference→compression_error, overfit→reasoning_error). Episodic traces as ground truth for counterfactual replay. The finding that raw trajectories remain competitive is evidence for CMD's Oracle Retrieval (episodic ground truth).
- **Open gap**: No automated method to detect when consolidation has degraded memory below usability threshold — this is precisely CMD's attribution role.
