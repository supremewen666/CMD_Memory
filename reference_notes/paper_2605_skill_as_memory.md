# Skill as Memory, Not Document: A Database-Native Substrate for Agent Skill Catalogs

- **来源**: Sarkar, Pranab. Zenodo preprint, 2026-05-11.
- **核心贡献**: Frames agent skill storage as "skill as memory, not document." Identifies three failure modes of document-first skill catalogs: (i) token burn from in-context structural metadata (919K tokens for 5K-skill full catalog, vs 369 tokens for top-K database-native); (ii) retrieval slowdown from filesystem organization (p50 87.3ms, p95 106.3ms at 5K scale); (iii) invalid-skill admission (0% for database-native vs 97% for YAML-parseable). Uses 5,000-skill public reproducible corpus. Does not claim novel database architecture — identifies a category mismatch.
- **关键概念**: skill as memory (not document), database-native substrate, token burn from structural metadata, invalid-skill admission, YAML-parseable acceptance, typed records with schema validation
- **CMD 相关性**: HIGH. Directly relevant to CMD's Failure Memory design. ECS records in Failure Memory are essentially corrective "skills" stored for future retrieval. The three failure modes map to CMD concerns: token burn → compact ECS (corrected-only mode); retrieval slowdown → keyword-based retrieval must be efficient; invalid-skill admission → Post-Repair Context Replay as quality gate. The "skill as memory" framing independently validates CMD's design choice to store Failure Memory as structured records, not free-text documents.
- **开放空白**: Does not address how skills are generated/validated — this is CMD's contribution (ECS generation via counterfactual replay + Post-Repair Context Replay validation). The two are complementary: CMD generates and validates ECS records; the database-native substrate stores and retrieves them efficiently.
