# Trojan Hippo: Weaponizing Agent Memory for Data Exfiltration

arXiv: 2605.01970, 2026-05-03

Authors: Debeshee Das, Julien Piet, Darya Kaviani, Luca Beurer-Kellner, Florian Tramèr, David Wagner

## Core Contribution

Systematic characterization of persistent memory poisoning attacks on LLM agents under a realistic threat model: attacker plants dormant payload via a single untrusted tool call, activates when user discusses sensitive topics (finance, health, identity), exfiltrates high-value personal data. Introduces (1) OpenEvolve-based adaptive red-teaming benchmark for stress-testing memory defenses, and (2) capability-aware security/utility analysis for reasoning about defense deployment.

## Key Concepts

- **Dormant payload**: Poisoned memory entry that stays inert until triggered by specific conversation topics
- **Single untrusted tool call**: More realistic than prior work requiring full memory write access — a single crafted email or web page suffices
- **OpenEvolve adaptive red-teaming**: Continuously refines attacks against defenses and memory backends
- **Capability-aware security/utility analysis**: Accounts for different usage profiles when reasoning about defense deployment

## CMD Relevance

1. **Validates `item_poisoned` (V2 label)**: Trojan Hippo provides the first systematic evidence that memory poisoning is a real, exploitable failure mode in deployed agent memory systems. This directly supports CMD's plan to add `item_poisoned` in V2.

2. **Attack surface = new failure mode**: The paper demonstrates that memory systems introduce a NEW class of failures beyond correctness — security failures where the memory entry is factually plausible but intentionally harmful. CMD's current taxonomy doesn't distinguish "wrong because of bug" from "wrong because of attack."

3. **Red-teaming benchmark → probe perturbation strategy**: The OpenEvolve adaptive red-teaming approach could inspire CMD's probe perturbation generation — automatically generating harder perturbations that stress-test specific pipeline labels.

4. **Connection to MAGE (2605.03228)**: Published 1 day apart, Trojan Hippo (attack) and MAGE (defense) together define the agent memory security subfield as of May 2026.

## Open Gap

CMD currently targets accidental/correctness failures, not adversarial ones. `item_poisoned` in V2 would need to distinguish "accidentally wrong" from "adversarially planted." Counterfactual replay for poisoning may require different oracles (e.g., "clean memory" oracle rather than "corrected memory").
