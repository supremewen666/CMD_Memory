# 2604.27045 - Dual-Stream Memory and Reconciliation for Health Coaching Agents

- **arXiv**: 2604.27045
- **Core Contribution**: Dual-Stream Memory Architecture separating patient narrative (current but biased) from structured clinical record/FHIR (validated but stale). Reconciliation Engine classifies discrepancies by type, severity, and FHIR resource. Detects 84.4% of discrepancies with 86.7% safety-critical recall. 13.6% error cascade traced to extraction loss.
- **Key Concepts**: Dual-stream memory, reconciliation engine, clinical discrepancy detection, extraction-loss cascade, memory validation against ground truth.
- **CMD Relevance**: Directly validates `premature_extraction_error` label — 13.6% error cascade from "clinical details lost during memory extraction from unstructured conversation" is empirical evidence for extraction-stage failure. Dual-stream pattern mirrors CMD's Verbatim Event Oracle (raw event stream) vs Extracted Memory comparison.
- **Open Gap**: Discrepancy detection is observational, not interventional. The reconciliation engine identifies mismatches but does not replay with corrected memory to verify repair.
