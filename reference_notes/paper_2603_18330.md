# MemArchitect: Policy-Driven Memory Governance Note

- arXiv: 2603.18330
- Core contribution: decouples memory lifecycle management from model weights via a governance layer enforcing decay, conflict resolution, and privacy control policies.
- Key concepts: policy-driven governance, memory decay policies, conflict resolution, privacy controls, managed vs unmanaged memory comparison.
- CMD relevance: memory governance policies are the production target that CMD's repair validation feeds into. MemArchitect's policy enforcement is where CMD's ECS repair guidance would be applied.
- Open gap: MemArchitect applies static policies; CMD provides the evidence to decide which policy to apply based on attribution results.
