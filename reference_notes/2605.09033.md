# 2605.09033 - ShadowMerge: Poisoning Attack on Graph-Based Agent Memory via Relation-Channel Conflicts

- **Core contribution**: Novel poisoning attack against graph-based agent memory exploiting relation-channel conflicts — poisoned relation shares query-activated anchor and canonicalized relation channel as benign evidence while carrying conflicting value. 93.8% average attack success rate.
- **Key concepts**: Relation-channel conflict; AIR pipeline (converts conflict into ordinary interaction that can be extracted/merged/retrieved); graph-memory poisoning surface; input-side defense insufficiency.
- **Method**: AIR pipeline: craft interaction where poison shares anchor + relation channel but conflicts on value → graph memory system extracts, merges, and retrieves it alongside benign evidence.
- **Results**: 93.8% average ASR on Mem0 across PubMedQA, WebShop, ToolEmu; +50.3pp over best baseline; negligible impact on unrelated benign tasks; representative input-side defenses insufficient.
- **CMD relevance**: Validates `injection_error` and future `item_poisoned` labels with concrete attack mechanism on graph memory. ShadowMerge's relation-channel conflict is a new failure signature CMD could detect: same anchor + relation channel but conflicting values. Graph-memory poisoning is structurally different from flat-memory injection — CMD may need graph-aware attribution for V1/V2.
- **Open gap**: Defense mechanism not proposed; detection of relation-channel conflicts in existing memory stores is open.
