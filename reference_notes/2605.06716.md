# 2605.06716 - From Storage to Experience: A Survey on the Evolution of LLM Agent Memory Mechanisms

- **Core contribution**: Three-stage evolutionary framework for agent memory: Storage (trajectory preservation) → Reflection (trajectory refinement) → Experience (trajectory abstraction). Identifies three drivers: long-range consistency, dynamic environments, continual learning.
- **Key concepts**: Storage stage (raw trajectory preservation); Reflection stage (trajectory refinement, summarization, filtering); Experience stage (trajectory abstraction, proactive exploration, cross-trajectory abstraction).
- **Frontier mechanisms**: Proactive exploration (agents actively seek experiences to fill memory gaps); cross-trajectory abstraction (learning patterns across multiple episodes, not just within-trajectory).
- **CMD relevance**: Places CMD in context: CMD-Audit operates primarily at Reflection→Experience boundary (diagnosing when refinement/abstraction fails). The Experience stage's proactive exploration is orthogonal — CMD is reactive diagnosis. The three-stage framework maps to CMD's error labels: Storage failures → `write_error`, Reflection failures → `compression_error`/`premature_extraction_error`, Experience failures → `reasoning_error`.
- **Open gap**: Framework is descriptive, not diagnostic — no method to detect which stage caused a specific failure.
