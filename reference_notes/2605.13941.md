# 2605.13941 EvolveMem: Self-Evolving Memory Architecture

- Core: Co-evolution of stored knowledge + retrieval mechanism; exposes full retrieval configuration as structured action space; LLM-powered diagnosis module reads per-question failure logs, proposes targeted configuration adjustments; guarded meta-analyzer with auto-revert-on-regression.
- Key: First system to evolve both what is stored AND how it is retrieved. Diagnosis → configuration adjustment → validation loop. Auto-revert prevents regression.
- CMD relevance: Structurally closest to CMD's diagnosis loop. EvolveMem diagnoses retrieval configuration; CMD diagnoses pipeline operations. Complementary layers: EvolveMem for retrieval config optimization, CMD for pipeline operation debugging.
- Gap: EvolveMem's diagnosis is about retrieval hyperparameters (scoring, fusion, generation policy); CMD's diagnosis is about which pipeline operation failed. EvolveMem doesn't do counterfactual replay.
