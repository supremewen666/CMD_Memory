# MemORAI: Memory Organization and Retrieval via Adaptive Graph Intelligence for LLM Conversational Agents

arXiv: 2605.01386, 2026-05-02

Authors: Hung Pham Van, Nguyen Manh Hieu, Khang Pham Tran Tuan, Nam Le Hai, Linh Ngo Van, Nguyen Thi Ngoc Diep, Trung Le

## Core Contribution

MemORAI is a graph-based memory framework with three innovations: (1) selective memory filtering with dual-layer compression to retain user-persona-relevant content, (2) provenance-enriched multi-relational graph tracking factual origins at the turn level, (3) query-adaptive subgraph retrieval with Dynamic Weighted PageRank using query-conditioned edge weighting.

## Key Concepts

- **Dual-layer compression**: First layer filters irrelevant turns, second compresses retained content — addresses `compression_error`
- **Provenance-enriched graph**: Multi-relational edges track which conversation turn each fact originated from — enables `premature_extraction_error` diagnosis
- **Dynamic Weighted PageRank**: Query-conditioned edge weighting makes retrieval query-adaptive rather than uniform — addresses `retrieval_error`
- **Selective memory filtering**: Only stores user-persona-relevant content, discarding noise

## Key Results

- State-of-the-art on LOCOMO and LongMemEval benchmarks
- Demonstrates that selective storage + enriched representation + adaptive retrieval are all essential

## CMD Relevance

1. **SOTA on CMD's V1 target benchmarks**: MemORAI achieves SOTA on the same LOCOMO and LongMemEval benchmarks that CMD targets for V1 real-data evaluation. It should be cited as both related work AND as the memory system to beat in V1.

2. **Provenance tracking validates CMD's approach**: MemORAI's turn-level provenance tracking independently validates CMD's need for evidence chains — knowing WHERE a fact came from is prerequisite for attributing WHICH pipeline step failed. MemORAI tracks provenance for retrieval quality; CMD uses provenance for failure attribution.

3. **Three innovations → three CMD labels**: Dual-layer compression → `compression_error`, provenance graph → `premature_extraction_error`, query-adaptive retrieval → `retrieval_error`. MemORAI's design choices directly reflect the failure taxonomy.

4. **Graph-based system adds V1 relevance**: As a graph memory system, MemORAI is also relevant for testing CMD's `graph_error` label (V1).

## Open Gap

MemORAI improves memory performance but provides no failure diagnosis mechanism. When it fails, there's no way to know whether the failure was in filtering, compression, provenance tracking, or retrieval. CMD's counterfactual replay is the diagnostic layer that MemORAI lacks.
