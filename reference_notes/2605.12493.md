# 2605.12493 - LongMemEval-V2: Evaluating Long-Term Agent Memory Toward Experienced Colleagues

- **Core contribution**: Extends LongMemEval with 451 manually curated questions over 5 core memory abilities for web agents: static state recall, dynamic state tracking, workflow knowledge, environment gotchas, premise awareness. Up to 500 trajectories and 115M tokens.
- **Key concepts**: Context gathering formulation (memory systems consume history, return compact evidence for QA); AgentRunbook-R (RAG with knowledge pools); AgentRunbook-C (coding agent in augmented sandbox).
- **Method**: Two memory methods — RAG-based with raw state/event/strategy pools vs coding agent that writes code to query evidence.
- **Results**: AgentRunbook-C achieves 72.5% accuracy vs strongest RAG baseline 48.5% and coding agent baseline 69.3%. Substantial room for improvement remains despite Pareto frontier advancement.
- **CMD relevance**: LME-V2's environment gotchas and workflow knowledge categories are natural probe case sources for CMD V1 real-data requirement. The context gathering formulation mirrors CMD's memory-as-evidence model. AgentRunbook-C's coding-agent approach could serve as oracle retriever for CMD.
- **Open gap**: No memory failure diagnosis or repair in benchmark protocol — evaluates retrieval quality but not failure attribution.
