# ADAM: Data Extraction Attack on Agent Memory Note

- arXiv: 2604.09747
- Core contribution: privacy attack on LLM agent memory modules using data distribution estimation and entropy-guided query strategies, achieving up to 100% attack success rate.
- Key concepts: adaptive querying, data distribution estimation, entropy-guided queries, memory privacy vulnerability.
- CMD relevance: ADAM demonstrates that agent memory is a high-value attack surface — validates CMD's injection_error label and the need for Subagent Judge Monitor's leak-safe boundaries. Memory contents must be protected from extraction, not just injection.
- Open gap: ADAM is attack-only; CMD's injection_error attribution and A-MemGuard-style defense complete the picture.
