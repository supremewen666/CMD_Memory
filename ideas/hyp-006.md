# Hypothesis 006

## 假设

CMD 会比 subagent judge 更适合 agent-memory failure attribution，因为 CMD 生成的是 intervention-grounded labels，而 subagent judge 主要生成 post-hoc explanations。

## 推理过程

启发式 evidence recall 只能判断 gold evidence 是否被取回，但无法区分写入失败、压缩失真、路由错误、注入格式误导、图扩展干扰和最终推理失败。subagent judge 能读取更多 trace，也能给出更像人的解释，但如果它没有实际执行 counterfactual replay，它仍然只是对同一条失败轨迹做观察性解释。

CMD 的关键新意是把每一种 memory operation 变成可干预变量：oracle write、oracle compression、route oracle、oracle retrieval、Verbatim Event Oracle、Injection-Oracle、graph intervention、safety intervention、evidence-given reasoning。哪一个干预带来最大恢复增益，就给出相应归因。这样标签不仅能和合成扰动 ground truth 对齐，还能直接绑定 repair action。
