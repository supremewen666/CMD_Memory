# Hypothesis 011

## 假设

D-MEM 的 Critic Router（Surprise + Utility 评分）可以作为 CMD Subagent Judge Monitor 的轻量级前置过滤器，在不触发完整反事实重放的情况下过滤低风险案例。

## 推理过程

D-MEM (2603.14597) 用 Reward Prediction Error 门控信号决定是否触发昂贵的 O(N) 知识图谱重组，80% token 节省来自低 RPE 案例被 O(1) 缓冲处理。CMD 面临类似问题：Subagent Judge Monitor 应该对每个失败任务都触发反事实重放，还是在重放前先做低成本异常预判？

如果 CMD 采用类似的 Surprise/Utility 快速评分（评估失败案例与 Memory Item 之间的证据落差），可以在不运行完整 V0 Replay Portfolio 的情况下：
1. 跳过明显不是记忆管线问题的案例（如用户输入模糊）；
2. 对高证据落差案例优先分配重放预算。

这与 CMD 当前 Subagent Judge Monitor 的定位（leak-safe，可触发重放但不输出最终标签）兼容：RPE 预过滤器位于 Monitor 之前，做 caseness 判断。
