# Hypothesis 005

## 假设

CMD 需要加入 Verbatim Event Oracle：许多看似 retrieval failure 的 agent-memory 错误，真实原因是写入或压缩阶段过早抽取，导致未来查询所需证据已经不可恢复。

## 推理过程

Storage Is Not Memory 明确指出，ingestion 阶段丢掉的内容在 retrieval 阶段无法恢复。这会让后续调试看起来像“检索没找对”，但实际是 memory item 或 memory schema 从一开始就没有保留可恢复证据。

MEMTIER 和 BudgetMem 一类工作强调 tier/routing/retrieval 的重要性，但如果原始事件已经被抽象损坏，继续优化 retrieval weight 可能只能在错误表征中搜索。Governed Collaborative Memory 又提醒我们，修正路径需要保留 provenance 和 version lineage。因此 CMD 应先比较 raw event replay 与 extracted memory replay，再决定 blame 是 retrieval、compression、write，还是 premature extraction。
