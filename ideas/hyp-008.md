# Hypothesis 008

## 假设

CMD 的 V0 必须加入 Post-Repair Context Replay；否则系统只能证明 attribution 或生成纠错建议，不能证明纠错后的上下文真的恢复了原失败任务。

## 推理过程

现有 prototype / PRD / TDD 已经覆盖 baseline failure、counterfactual replay、attribution、ECS、targeted fix 和 future Failure Memory retrieval。但这些步骤之间缺少一个关键验证：把 ECS 生成的 corrected memory、repair guidance 和 repaired evidence block 重新拼成上下文，对同一个原始 query 再跑一次。

如果不做这个复测，CMD 的贡献会停在“知道错在哪里”和“建议怎么改”。加入 Post-Repair Context Replay 后，CMD 可以进一步报告 repair_success、post-repair answer score、post-repair evidence score，并与 generic hard-case update 对比。这样论文主张会从 attribution 扩展到 repair-validated diagnosis。
