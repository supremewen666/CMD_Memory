# Hypothesis 007

## 假设

CMD 应采用 audit-module-first、skill-adapter-second 的形态：核心方法作为外部审计模块证明归因能力，部署时再通过 skill adapter 把 ECS 转成可执行修复策略。

## 推理过程

CMD 的研究贡献来自 counterfactual replay。它需要外部控制 memory write、compression、routing、retrieval、injection、graph、safety 和 reasoning 等环节，并通过答案恢复增益判断失败来源。这种能力更像审计器或 debugger，而不是 agent 内部普通 skill。

但 skill 形态也不能丢。Failure Memory 中的 Error-Cause-Solution 本质上就是可复用经验：后续相似任务应该调用一个 skill adapter，只注入 `corrected_memory + repair_guidance`，并把 hard-case labels 反馈给 MemSkill-style skill evolution。

因此最稳的论文叙事是：CMD-Audit 负责可验证的诊断；CMD-Skill Adapter 负责可部署的修复。
