# Hypothesis 015

## 假设
CMD's counterfactual replay over episodic traces can serve as both attribution engine (which operation degraded evidence) and cascade repair validation gate (do repaired descendants preserve decision-relevant distinctions), forming an episodic-trace anchored cascade repair pipeline when paired with MEMOREPAIR's barrier-first repair contract.

## 推理过程

**Cross-paper convergence.** Three Day 4 papers independently converge on episodic traces as ground truth for memory operations: 2605.12978 shows episodic traces remain competitive with consolidators and agents choose episodic-first policies; 2605.10870 shows memory value is preserving decision-relevant distinctions (which episodic traces inherently contain); 2605.07242 MEMOREPAIR shows cascade repair depends on retained pre-event support — which is structurally equivalent to episodic traces before the degradation event.

**The attribution→cascade gap.** MEMOREPAIR formalizes the cascade update problem and provides optimal repair selection (s-t min-cut), but requires manual identification of the invalidated root set F. CMD's counterfactual replay provides exactly this: operation-level attribution identifies which memory operation caused the degradation, which maps to MEMOREPAIR's root set. No existing method bridges attribution and cascade repair.

**Validation alignment.** MEMOREPAIR's validation checks (schema validity, execution compatibility, task/safety behavior) are syntactic/behavioral. CMD's Post-Repair Context Replay adds semantic validation: does the repaired context recover the original task success? This is a natural upgrade to MEMOREPAIR's validation step.

**Decision-centric grounding.** 2605.10870's decision-centric rate-distortion framework provides a principled measure: repaired memory succeeds if it preserves decision-relevant distinctions from the original episodic traces. This gives a mathematical grounding to Post-Repair Context Replay's `recovered/partial/failed` assessment.

## 自评
- 新颖性: 3 — Integration is novel (no existing attribution+cascade pipeline), but components exist independently; episodic anchoring is distinct from both CMD's current design (counterfactual replay) and MEMOREPAIR (provenance tracking)
- 可行性: 4 — CMD's counterfactual replay already uses episodic traces via Verbatim Event Oracle. MEMOREPAIR's s-t min-cut is exact and implementable. Main barrier: influence provenance tracking is not yet in CMD's infrastructure (V1/V2 scope)
- 影响力: 4 — If validated, this turns CMD from a standalone attribution tool into the frontend of a complete detection→diagnosis→cascade-repair→validation pipeline. The episodic anchoring principle connects three independently validated findings into a unified architecture.
