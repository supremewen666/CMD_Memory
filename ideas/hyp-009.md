# Hypothesis 009

## 假设

CMD 的 V0 probe 可以借鉴 MEMAUDIT 的 package-oracle 思路，把每个失败样例设计成可审计 package，从而减少对 LLM judge 的依赖，并让 write/compression/premature extraction 标签更可验证。

## 推理过程

MEMAUDIT 把长期记忆写入评估转成固定 experience stream、candidate memory representations、semantic evidence units、future-query requirements 和 budget 的有限优化问题。这与 CMD V0 的目标高度贴合：CMD 需要在合成扰动中知道证据是否被写入、是否被压缩丢失、是否在 extraction 阶段变得不可恢复。

如果每个 CMD probe case 都包含 raw events、candidate extracted memories、gold evidence units、future query requirements 和 expected repaired context，那么 Oracle Write、Oracle Compression、Verbatim Event Oracle 和 Post-Repair Context Replay 都可以用同一 package 结构评估。这样 CMD-Audit 会更像一个可复现 benchmark，而不是一组松散 prompt tests。
