## Issue 0019: Subagent Scoring — Grill-with-Docs Design Refinements

### Design Decisions (grill-with-docs session)

After A/B split, 8 architectural questions resolved:

1. **AnswerVerifier strategy**: Both EvidenceVerifier and AnswerVerifier implemented in Phase B. EvidenceVerifier active (replaces `evidence_recall_from_text`). AnswerVerifier implemented but not wired — deferred to Decision B when real agent output exists.

2. **`scorer` parameter contract**: `Callable[[tuple[GoldEvidence, ...], str], float]` — aligns with `evidence_recall_from_text` signature. EvidenceVerifier's atomic `(FACT, TEXT)` adapted internally by `SubagentScorer.score_evidence()`. `evidence_recall_from_text` preserved as default/fallback (`scorer=None`).

3. **LLMJudgeBaseline input boundary**: Observable post-hoc trace artifacts only (raw_events, extracted_memory, baseline answer, query). NO gold_label, ptype, gold_evidence, or gold_answer. Distinction from Subagent Judge Monitor: Monitor is enum-locked `anomaly_reason`; LLMJudgeBaseline outputs `DiagnosisPrediction` label.

4. **hooks.py mount point**: Pure validation functions exposed in hooks.py; SubagentScorer internally enforces them at subagent call boundary. External callers cannot bypass.

5. **ollama transport**: Subagent and main agent share one LLM API — `llm_client.py` is provider-agnostic, not hardcoded to ollama. Output parse failures handled inside SubagentScorer (retry once with stricter prompt, then fallback to phrase-matching).

6. **New module: `llm_client.py`**: Provider-agnostic LLM API client (`generate(prompt) -> str`). Shared dependency for both LLMJudgeBaseline (Phase A) and SubagentScorer (Phase B).

7. **New module: `subagent_runner.py`**: Claude Code-style subagent runner. Minimal `run(system_prompt: str, user_message: str) -> str`. Isolated context window, dedicated system prompt, no tool access, returns result only.

8. **File creation order (dependency-first)**: `llm_client.py` → `llm_judge.py` → `subagent_runner.py` → `llm_scoring.py` + `hooks.py`. Avoids Phase A accumulating inline ollama call debt that would be deleted in Phase B.

### Module Architecture

```
llm_client.py          ← provider-agnostic LLM API (Phase A/B shared)
llm_judge.py           ← LLMJudgeBaseline (Phase A, uses llm_client directly)
subagent_runner.py     ← Claude Code-style subagent (Phase B, uses llm_client)
llm_scoring.py         ← EvidenceVerifier (active) + AnswerVerifier (deferred) + SubagentScorer
hooks.py               ← pure validation functions (enforced internally by SubagentScorer)
```

## Pre-CMD Gate Architecture Design (2026-05-21)

### Architecture Overview

Refactored PrefixGuard + RPE + Subagent Judge Monitor into a unified **Pre-CMD Gate**: a two-layer self-supervised gate system that sits before CMD diagnosis. ECC hook pattern: event-driven, composable, declarative registration, per-adapter instances.

```
Layer 0 (Pre-emptive): PrefixGuard + RPE at memory.post_retrieve  →  trigger CMD early
Layer 1 (Reactive):     Post-Answer Gate at post_reason            →  catch reasoning errors
  ↓ (CMD Diagnosis Layer triggers only when gate fires)
```

### Key Design Decisions

1. **Two-layer self-supervised gate**: Layer 0 catches 10/11 labels at `post_retrieve`; Layer 1 (Post-Answer Gate) catches `reasoning_error` (only detectable post-answer) + Layer 0 false negatives. Zero gold dependency for online deployment.

2. **Online self-supervised signals** (replacing gold-dependent offline signals):
   - Deterministic rules: `empty_context`, `safety_trace`, `truncation`, `cross-item_contradiction`
   - Embedding-based semantic surprise: `1.0 - cosine_sim(embed(query), embed(memory))`
   - Agent confidence: logprob entropy, token probability
   - Historical recovery rates per replay type (V2 calibration)

3. **Per-adapter PreCmdGate** (NOT global singleton): Each adapter instantiates its own `PreCmdGate`, registers hooks, fires events per operation.

4. **11 hook events**: `memory.pre_store`, `memory.post_store`, `memory.pre_compress`, `memory.post_compress`, `memory.pre_extract`, `memory.post_extract`, `memory.post_retrieve` (primary CMD trigger), `memory.pre_inject`, `memory.pre_reason`, `memory.post_reason` (Post-Answer Gate trigger), `memory.post_safety`.

5. **OnlineReplayRanker Protocol**: Separate from offline `compute_per_replay_ordering`. D-MEM-style Critic Router self-supervision: Surprise × Utility = RPE score. `record_outcome()` for V2 historical calibration.

6. **Subagent Judge Monitor → Post-Answer Gate**: NOT deprecated — refactored into Layer 1 with unique `reasoning_error` detection value. Catches what Layer 0 misses.

7. **Gold evidence dependency**: 4/11 labels need gold_evidence for counterfactual replay (information-theoretic bound: detecting missing content requires knowing what should exist). Online gate achieves 7-9/11 via self-supervision.

8. **Naming conventions**: `PreCmd` prefix (not generic "Hook"), `PreCmdGate` (not "HookManager"), `pre_cmd_gate.py` (not "hooks.py").
