### Architecture

```
post_retrieve_hook(query, retrieved_items)
  → empty_ctx? → trigger_cmd (reason: "empty_context")
  → utility_override? (agent_confidence < 1-UTILITY_OVERRIDE_THRESHOLD) → skip
  → rpe_above_threshold? (surprise × utility > RPE_THRESHOLD) → trigger_cmd
  → anomaly_above_threshold? (PrefixGuard weighted score > ANOMALY_THRESHOLD) → trigger_cmd
  → clean → skip
```

PrefixGuard signals (4 zero-gold): `empty_ctx`, `truncation` (11 patterns), `near_duplicate` (Jaccard), `low_count`.
RPE: `surprise = 1 - max(BM25(query, items))`, `utility = 1 - agent_confidence`.

## Decision 33 Update (2026-05-23): PrefixGuard 独立阶段移除

Discussion session 确认 PrefixGuard 独立阶段移除，简化为两阶段架构：

### 原因

- **truncation 11 pattern**：为手写猜测（PrefixGuard 论文 + memory system 行为推断），线上检索记忆为纯文本，不带 `[blocked]` 等元数据标签 → 结构性零召回
- **near_duplicate / low_count**：信息性信号而非确定性信号，更适合作为 RPE Judge 连续特征输入

### 变化

| 维度 | 旧 (3-stage) | 新 (2-stage) |
|------|-------------|-------------|
| 架构 | PrefixGuard → RPE Judge → fallback | empty_ctx 硬短路 → RPE Judge → fallback |
| 特征数 | 15 (5 全局 + 10 one-hot) | 19 (9 全局 + 10 one-hot) |
| 阈值数 | 3 (anomaly_hard + top_p + fallback) | 2 (top_p + fallback) |
| hook/ 包 | prefix_guard.py + rpe_judge.py + post_retrieve_hook.py + constants.py | rpe_judge.py + post_retrieve_hook.py + constants.py + __init__.py |
| PreCmdDecision.stage | "hard_trigger" \| "rpe_top_p" \| "clean" | "empty_ctx" \| "rpe_top_p" \| "clean" |
| PreCmdDecision 字段 | 含 anomaly_score, prefix_guard_signals | 移除 anomaly_score, prefix_guard_signals |

### 新增全局特征（4 个）

`near_duplicate` (Jaccard), `low_count` (item_count < 2), `safety_filter_blocked`, `is_graph_expanded`, `store_count`（原 5 全局 → 9 全局）
